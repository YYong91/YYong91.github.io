오늘 Claude Code 하나로 꽤 많은 작업을 처리했습니다.

Hugo 포트폴리오 사이트를 만들고, 배포하고, 다크모드 버그를 고치고, 네비게이션 간격을 조정했습니다. 거기에 Claude Code 플러그인까지 만들었습니다. 전부 하나의 대화에서 진행한 것입니다.

## 진행 과정

처음에는 포트폴리오 사이트 하나만 만들려고 시작했는데, 플러그인 아이디어가 떠올랐습니다. Claude Code 세션 데이터에서 자동으로 블로그 포스트를 생성해주는 플러그인입니다.

`executing-plans` 스킬을 사용해서 계획을 체크포인트 단위로 묶어 실행했는데, 생각보다 훨씬 편리했습니다. 문제가 발생했을 때 어디서 멈춘 것인지 바로 파악할 수 있기 때문입니다.

`brainstorming` 스킬로 플러그인 구조를 잡고, `writing-plans` 스킬로 구현 계획을 문서화하고, 실제 실행은 `executing-plans`로 진행했습니다. 스킬 조합이 잘 맞았습니다.

## dev-portfolio 플러그인

만든 플러그인 이름은 `dev-portfolio`입니다. 에이전트 5개, 커맨드 6개, 스킬 2개로 구성되어 있습니다.

핵심 아이디어는 이것입니다. Claude Code로 작업을 하면 세션 데이터가 남는데, `/portfolio wrap` 커맨드 하나로 블로그 포스트를 뽑아내는 것입니다. 실제로 이 포스트가 그 방식으로 만들어졌습니다.

`session-extractor` 에이전트가 세션 컨텍스트를 분석해서 JSON raw log를 생성하고, `til-writer`나 `claude-exp-writer` 같은 writer 에이전트가 그 데이터를 기반으로 마크다운 파일을 생성합니다.

## 사이트

Hugo + PaperMod 기본 셋업은 빠른 편이었지만, 다크모드 색상이 이상하게 나오는 문제가 있었습니다. CSS 변수 오버라이드로 해결했고, 네비게이션과 본문 사이의 색상 불일치도 수정했습니다.

GitHub Actions로 배포는 자동이기 때문에 커밋만 올리면 됩니다. 워크플로우 파일도 같이 생성했기 때문에 따로 설정할 것이 없었습니다.

커밋 11개, 497줄 추가, 파일 10개. 한 세션 치고는 상당한 양입니다.

## 느낀 점

스킬을 미리 잘 정의해두면 반복 작업이 확실히 줄어듭니다. `executing-plans`처럼 체크포인트가 있는 실행 방식이 긴 작업에서 특히 안정적이었습니다.

플러그인 자체가 자기 자신의 포스트를 작성하는 구조가 흥미로웠습니다.
