오늘 Claude Code 하나로 꽤 많은 걸 해버렸어요.

Hugo 포트폴리오 사이트를 만들고, 배포하고, 다크모드 버그 고치고, 네비게이션 간격 조정하고... 그것도 모자라서 Claude Code 플러그인까지 만들었어요. 전부 대화 하나에서.

## 어떻게 했냐면

처음엔 그냥 포트폴리오 사이트 하나 뚝딱 만들려고 시작했는데, 플러그인 아이디어가 떠올랐어요. Claude Code 세션에서 자동으로 블로그 포스트를 생성해주는 플러그인이요.

`executing-plans` 스킬이 있어서 계획을 체크포인트 단위로 묶어서 실행할 수 있었는데, 이게 생각보다 훨씬 편했어요. 뭔가 잘못됐을 때 어디서 멈춘 건지 바로 알 수 있으니까요.

`brainstorming` 스킬로 플러그인 구조 잡고, `writing-plans` 스킬로 구현 계획 문서화하고, 실제 실행은 `executing-plans`로. 스킬 조합이 생각보다 잘 맞았어요.

## dev-portfolio 플러그인

만든 플러그인 이름은 `dev-portfolio`예요. 에이전트 5개, 커맨드 6개, 스킬 2개로 구성되어 있어요.

핵심 아이디어는 이거예요. Claude Code로 작업을 하면 세션 데이터가 남잖아요. 그걸 `/wrap` 커맨드 하나로 블로그 포스트로 뽑아내는 거예요. 지금 이 포스트가 실제로 그렇게 만들어진 거고요.

```
/wrap
```

이거 하면 `session-wrap` 플로우가 돌아가면서 세션에서 뭘 했는지 분석하고, 글쓰기 스타일 가이드에 맞춰서 Hugo 마크다운 파일을 생성해요.

## 사이트 쪽에서

Hugo + PaperMod 기본 셋업은 빠른데, 다크모드 색상이 이상하게 나오는 문제가 있었어요. CSS 변수 오버라이드로 고쳤고, 네비게이션 아이템 간격도 조금 손봤어요.

GitHub Actions로 배포는 자동이라서 커밋만 올리면 끝이에요. 이 부분은 Claude Code가 워크플로우 파일도 같이 잡아줘서 따로 손 안 댔어요.

커밋 11개, 497줄 추가, 파일 10개. 한 세션 치고는 꽤 나왔어요.

## 느낀 점

스킬을 미리 잘 정의해두면 반복 작업이 확실히 줄어요. `executing-plans`처럼 체크포인트 있는 실행 방식이 긴 작업에서 특히 안정적이었고요.

플러그인 자체가 자기 자신의 포스트를 쓰는 구조가 좀 재밌었어요. 메타적이랄까.
